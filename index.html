The TLS Archiver Project

The TLS Archiver

More and more X.509 certificates are delivered and generated every month, with its part of weird or mis-issued certificates. Let's archive all of them!

Check out the repository

To get started, prepare first your environment:

mkdir TLSArchiver && cd TLSArchiver
git clone https://github.com/tlsarchiver/infra.git
cd infra
docker-compose up

Then, execute the archiver:

docker run --rm -it --link=tlsa-db --network=infra_default -e ARCHIVER_DBPASSWORD=kakoo -e ARCHIVER_DBHOST=tlsa-db tlsarchiver/archiver /archiver

So many certificates

TLS is taking a growing place in the modern internet, be it for protecting HTTP or for other services such as email (SMTP, POP, IMAP). Issuing a certificate has become a usual action for administrators and developers, and the number of certificates has grown drastically. The TLS Archiver Project aims at downloading all the available X.509 certificates available on the internet, and to conduct analysis on them... in the hope to find interesting things ;)

State of the project

The project is under heavy development, and cannot be considered as stable. Pull Requests are welcome!

Goals

The first goal is to collect as many certificates as possible, by using a wide panel of sources. The second goal is to process them and conduct detailed analysis on them to discover re-usage, mis-issuance, and other funky (or scary) things, and to produce some nice analytics on the state of the certificate landscape.
